## Кейс 1: Switchback-тест для доставки еды

### 🎯 Постановка задачи
Компания по доставке еды планирует изменить время действия специальных скидок:
- **Было**: 12:00 – 14:00
- **Стало**: 18:00 – 20:00

Необходимо оценить эффективность нового расписания, исключив влияние дней недели и внешних факторов.

### 🔧 Решение

**Тип теста**: Switchback (кроссоверный временной эксперимент)

**Ключевые особенности дизайна**:
- Ежедневное переключение условий (день — тест, день — контроль)
- Стратифицированная рандомизация по дням недели
- Длительность: 2–4 недели (14–28 дней)
- Учёт carryover-эффектов через лаговые переменные

**Метрики**:
- **Primary**: Чистая маржа за день, объём заказов в тестовом окне
- **Secondary**: Конверсия, средний чек, retention D7/D30
- **Guardrail**: Нагрузка на курьеров, отмены заказов

**Статистический анализ**:
- Панельная регрессия с фиксированными эффектами по дням недели
- Кластеризация стандартных ошибок на уровне дней
- Контроль внешних факторов (погода, праздники)

### 📁 Файлы кейса
- `switchback_analysis.ipynb` — основной ноутбук с анализом
- `outlier_detection.py` — поиск выбросов методом IQR

## Кейс 2: Interleaving-тест для рекомендательных алгоритмов

### 🎯 Постановка задачи
Кинопоиск хочет протестировать 3 разных алгоритма рекомендаций видео. Необходимо определить лучший алгоритм с минимальными затратами времени и трафика.

### 🔧 Решение

**Тип теста**: Interleaving (Team Draft метод)

**Алгоритм чередования**:
```
Раунд 1: Алгоритм A → позиция 1
         Алгоритм B → позиция 2
         Алгоритм C → позиция 3
         
Раунд 2: Алгоритм A → позиция 4
         Алгоритм B → позиция 5
         Алгоритм C → позиция 6
... и так далее
```

**Преимущества перед классическим A/B-тестом**:
- Сравнение внутри одного пользователя (не между разными)
- В 10–100 раз выше статистическая мощность
- Контроль position bias через ротацию стартовой позиции
- Достаточно 2–3 дней вместо 1–2 недель

**Метрики**:
- **Primary**: CTR по алгоритму, AHC (Average Highest Click)
- **Secondary**: Watch Time, Completion Rate, Session Depth
- **Guardrail**: Негативные фидбэки, отписки

### 📊 Расчёт длительности теста

**Исходные данные**:
- DAU: 5 млн человек
- Конверсия в клик: 14%
- AHC: 4.4

**Результат**: Для обнаружения эффекта в 1 п.п. достаточно **2–3 дней** при заданном трафике.

### 📁 Файлы кейса
- `interleaving_test.ipynb` — расчёт метрик и длительности теста
- `team_draft_algorithm.py` — реализация алгоритма чередования
